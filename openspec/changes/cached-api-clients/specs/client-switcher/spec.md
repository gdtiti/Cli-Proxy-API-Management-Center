# client-switcher 规范

## 新增需求

### 需求:用户可以快速切换客户端

系统应当提供快速切换功能，允许用户在已保存的客户端配置之间快速切换。

#### 场景:通过下拉菜单切换客户端

- **当** 用户点击客户端选择下拉菜单并选择一个配置
- **那么** 系统自动填充该配置的服务器地址和管理密钥，并尝试连接

#### 场景:显示当前活动客户端

- **当** 用户已连接到一个客户端
- **那么** 系统在界面上显示当前活动的客户端名称

#### 场景:快捷键切换客户端

- **当** 用户按下预设的快捷键（如 Ctrl+1, Ctrl+2 等）
- **那么** 系统切换到对应的客户端配置

### 需求:切换时自动验证连接

系统应当在切换客户端时自动验证连接是否有效。

#### 场景:切换到有效客户端

- **当** 用户切换到有效的客户端配置
- **那么** 系统成功连接并显示连接状态

#### 场景:切换到无效客户端

- **当** 用户切换到无效或过期的客户端配置
- **那么** 系统显示连接错误，允许用户重新输入凭据

### 需求:记住上次选择的客户端

系统应当记住用户上次选择的客户端配置。

#### 场景:应用启动时恢复上次客户端

- **当** 用户重新打开应用
- **那么** 系统自动选择并填充上次使用的客户端配置
