# 变更提案：cached-api-clients

## 为什么

当前 CLI Proxy API 管理中心每次连接都需要手动输入服务器地址和管理密钥，用户在同时使用多个 API 地址（如本地开发环境、测试环境、生产环境）时，需要频繁切换和重新输入连接信息，导致效率低下。

## 变更内容

1. **新增多客户端缓存功能**：支持在本地保存多个 API 客户端配置，每个配置包含服务器地址、管理密钥和自定义名称
2. **新增快速切换功能**：提供下拉菜单或快捷键快速在不同客户端配置之间切换
3. **新增客户端管理界面**：允许用户添加、编辑、删除和重命名客户端配置
4. **持久化存储**：将客户端配置存储在本地配置文件中，重启后保持

## 功能 (Capabilities)

### 新增功能

- **multi-client-cache**: 支持本地缓存多个 API 客户端配置，每个配置包含服务器地址、管理密钥和显示名称，支持快速切换
- **client-switcher**: 提供客户端快速切换 UI 组件，支持下拉选择和快捷键切换

### 修改功能

（无）

## 影响

- **前端**：新增客户端管理页面/组件，修改登录页面增加客户端选择功能
- **后端**：无需修改
- **存储**：新增本地配置文件存储客户端列表
- **配置**：新增客户端缓存相关配置项
