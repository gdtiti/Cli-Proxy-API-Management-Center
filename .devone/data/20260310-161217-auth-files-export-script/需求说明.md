## 问题定义

- 当前问题: 希望通过一个可重复执行的脚本，从指定的管理端（`/management.html` 对应的同源 API）批量下载所有认证文件，并保存到本地目录中，便于备份/迁移/审计。
- 影响对象: 管理端运维/管理员（持有管理端凭证的人）；导出的文件包含敏感凭证，需要明确风险与操作边界。
- 触发背景:
  - 当前页面可在浏览器里逐个/批量导出，但在文件数量较多时可能受浏览器资源限制影响；
  - 希望在本地用脚本一键导出到目录，便于后续打包、同步或纳入备份流程。
- 为什么现在处理: 认证文件数量上来后，手工下载不可持续；脚本导出能形成“可复现、可记录、可自动化”的备份链路。

## 目标与成功标准

- 目标 1: 提供一个脚本，可从指定管理端批量导出认证文件到本地目录（一个文件对应一个 `.json`）。
- 目标 2: 导出过程可控、可观察：支持限并发、失败不中断、输出汇总，并避免把敏感凭证写入代码仓库。
- 成功标准:
  - 指定 `--base` 与 `--out` 后脚本可运行完成；
  - 本地目录生成与服务端列表一致的文件集合（默认排除 `runtimeOnly`）；
  - 失败时给出清晰错误与“成功/失败/跳过”统计；出现部分失败时仍保留成功导出文件；
  - 脚本与文档不包含明文凭证；凭证必须通过环境变量或运行时参数注入（默认环境变量）。

## 范围边界

### 本次必须完成

- 提供一个可执行脚本（推荐 Node.js，跟随当前仓库依赖），支持：
  - 从 `GET /v0/management/auth-files` 拉取文件列表；
  - 对列表中的每个 `name` 调用 `GET /v0/management/auth-files/download?name=...` 下载；
  - 将下载内容保存到本地输出目录；
  - 限并发下载（默认 4，可配置）；
  - 失败不中断（记录失败列表，最后汇总退出码与统计）。
- 安全边界:
  - 默认不导出 `runtimeOnly` 文件（保持与管理端 UI 批量导出一致的默认语义）；
  - 处理潜在的文件名注入/路径穿越（禁止 `..`、路径分隔符等）。
- 提供最小可复验的验收步骤与“不要泄露凭证”的操作指南。

### 本次明确不做

- 不做服务端打包 ZIP、不改后端接口。
- 不做导出文件加密/脱敏（若需要，作为后续可选任务）。
- 不做增量同步、断点续传、hash 校验（本次以“全量导出+失败可重跑”为主）。

## 验收真相陈述

- [REQ-001] 脚本可通过 `--base http://<host>:<port>` 连接到管理端同源 API，并能成功获取认证文件列表（`/v0/management/auth-files`）。
- [REQ-002] 脚本可将列表中的认证文件逐个下载并保存到本地目录 `--out <dir>`，文件名与服务端 `name` 一致（必要时做安全化），内容为服务端返回的原始文件内容。
- [REQ-003] 脚本默认排除 `runtimeOnly` 文件；提供显式选项可覆盖该默认行为（若确有需要）。
- [REQ-004] 脚本不会在代码、日志或任务文档包里写入明文凭证；凭证只能通过环境变量（例如 `CPA_MANAGEMENT_KEY`）或运行时参数提供。
- [REQ-005] 导出过程具备资源控制：限并发下载，避免一次性并发过高导致 `net::ERR_INSUFFICIENT_RESOURCES` 一类问题；并输出汇总统计（成功/失败/跳过）。

## 约束与依赖

- 技术约束:
- 依赖管理端已有接口（与 Web UI 使用的接口一致）：
  - `GET /v0/management/auth-files`
  - `GET /v0/management/auth-files/download?name=...`
- 鉴权方式: `Authorization: Bearer <managementKey>`。
- 运行环境: 本机 Node.js（版本以当前项目可运行的 Node 为准），以及能访问目标服务的网络。
- 业务约束:
- 认证文件属于高敏数据；导出后必须由操作者自行妥善保管（目录权限、备份策略、避免误提交）。
- 外部依赖:
- 需要一个有效的管理端凭证（管理端 key）。
- 目标服务可访问（网络、端口、ACL、反向代理等）。

## 风险与待确认项

- 风险:
- 安全风险: 导出会把敏感凭证落地到本地磁盘；需要默认输出目录提示与 `.gitignore` 防呆，避免误提交。
- 稳定性风险: 文件数量大时导出耗时较长；需要限并发与清晰进度/统计。
- 兼容性风险: 认证文件名可能包含特殊字符；脚本必须做跨平台文件名安全化与路径穿越防护。
- 待确认（默认假设）:
  - “导出全部”语义 = 导出所有可下载认证文件（默认排除 `runtimeOnly`）。
  - 导出目录由用户通过 `--out` 指定；脚本不擅自写入 `.devone/`。
