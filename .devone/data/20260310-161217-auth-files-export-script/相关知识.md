## 术语与定义

- 管理端页面: `management.html`，通过 API 服务同源提供的单页管理界面。
- 管理端 API: 以 `/v0/management/*` 为前缀的一组管理接口。
- 认证文件（Auth File）: 管理端维护的凭证 JSON 文件，列表项 `name` 作为下载与保存的唯一标识。
- `runtimeOnly`: 运行态临时文件标记（通常不应参与导出/删除等批量操作）。

## 调研结论

- 结论 1: 脚本导出不需要解析 `management.html` 的 DOM；直接复用同源管理 API 即可完成“列出 + 下载保存”闭环。
- 结论 2: 关键风险不在“接口有没有”，而在“并发/资源控制 + 文件名安全 + 凭证不落盘不入库”的工程防线。

## 风险假设

- 假设:
- 文件数量可能达到几十到上百；需要限并发与失败可重跑。
- `name` 可能包含邮箱等字符（如 `@`），但原则上应可作为文件名；仍需防止路径穿越（`../`、`\\`、`/`）。
- 验证方式:
  - 先对 1~3 个文件做小规模导出验证（确认鉴权与保存正确）。
  - 再对“全部文件”做一次导出验证（确认限并发有效，且统计准确）。
  - 通过对比“列表数量”和“本地保存文件数量”做一致性核对。
