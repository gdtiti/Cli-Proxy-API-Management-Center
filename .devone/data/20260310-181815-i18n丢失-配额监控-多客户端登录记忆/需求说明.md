## 问题定义

- 当前问题：
  1. 侧边栏菜单「监控」的多语言缺失，当前会显示为空或显示 i18n key（例如 `nav.monitor`）。
  2. 侧边栏菜单「配额/配额管理」用户反馈也存在多语言丢失（需要在本地复现并确认具体表现）。
  3. 之前开发过的「多客户端登录记忆 / 快速切换」功能在当前版本不再出现，导致只能记住单一连接信息。

- 影响对象：
  - 需要同时管理多个环境（本地/测试/线上）的管理中心用户。

- 触发背景（初步定位）：
  - `MainLayout.tsx` 的导航项使用 `t('nav.monitor')`，但 `src/i18n/locales/*` 的 `nav` 节点目前缺少 `monitor` 键。
  - 多客户端相关 store/组件已存在（`useClientCacheStore`、`ClientSelector`、`ClientManagementModal`、`useClientKeyboardShortcuts`），但未接入任何页面，导致功能“看起来丢失”。

- 为什么现在处理：
  - 属于明显的体验回归，影响日常运维效率；且用户已明确反馈。

## 目标与成功标准

- 目标 1：修复侧边栏菜单「监控」「配额/配额管理」的多语言显示问题。
  - 成功标准：语言切换为 `zh-CN / en / ru` 时，侧边栏对应菜单均显示为译文，不出现空白或 i18n key（例如 `nav.monitor`）。

- 目标 2：恢复“多客户端登录记忆”（本地持久化）与“快速切换”（至少在登录页可快速选中并登录）。
  - 成功标准：
    - 登录页能从已保存的客户端列表中选择（下拉），选择后自动填充 `apiBase` 与 `managementKey`。
    - 支持在 UI 中新增/编辑/删除客户端配置；刷新页面后仍保留（持久化）。
    - 管理密钥不以明文写入 `client-cache` 的持久化 JSON（密钥仅存于 `secureStorage` 加密项）。

- 目标 3：登录后也能快速切换客户端（至少提供 Header 快速切换入口；快捷键为可选增强）。
  - 成功标准：
    - 登录后在 Header 可选择已保存客户端并触发“重新连接 + 刷新当前页数据”。
    - 切换失败时不会把用户踢回登录页，仍保持在原会话/原连接上，并提示失败原因。

## 范围边界

### 本次必须完成

- 修复侧边栏菜单 i18n：为 `nav.monitor` 补齐 `zh-CN/en/ru` 三种语言的翻译；并复核 `nav.quota_management` 在三种语言下均可正常显示。
- 恢复多客户端登录记忆（登录页最小闭环）：

- 登录后快速切换（Header 闭环）：
  - 在 Header 提供客户端切换控件（下拉 + 管理入口）。
  - 选择后自动重连并触发当前页面刷新（通过现有 Header refresh 机制），避免显示旧环境数据。
  - 切换失败时回退到原连接，不改变当前活动客户端。
- 安全与持久化：管理密钥仅存于 `secureStorage` 加密项；`client-cache` 的 persist JSON 中不包含明文密钥。
- 文案补齐：补齐 `client_selector.*`、`client_management.*` 等新增/缺失的 i18n keys（至少 `zh-CN/en/ru`）。

## 验收真相陈述

- [REQ-001] 在语言切换为 `zh-CN / en / ru` 时，侧边栏菜单「监控」「配额管理」均显示为译文，不出现空白或 i18n key（例如 `nav.monitor`）。

- [REQ-002] 登录页具备“多客户端登录记忆”的可操作入口：
  - 已有保存客户端时，出现客户端下拉选择；选择后会自动填充 `apiBase` 与 `managementKey`。
  - 提供客户端管理入口，支持新增/编辑/删除客户端，并可选择为当前活动客户端。

- [REQ-003] 多客户端配置具备持久化与安全边界：
  - 刷新页面后客户端列表仍在（来自 `client-cache` persist）。
  - `client-cache` 的持久化 JSON 不包含明文 `managementKey`（值应为空字符串）。
  - `managementKey` 仅存于 `secureStorage` 的加密项（例如 `client-cache-<id>-key`），并能被 `getClients()/getClientById()` 正确取回。

- [REQ-004] 新增/补齐的 UI 文案在 `zh-CN/en/ru` 三种语言下均有对应 i18n key，不出现 `client_selector.*` / `client_management.*` 之类的 key 回显。

- [REQ-005] 登录后可在 Header 快速切换客户端：
  - 通过下拉选择切换到已保存客户端配置，切换后会重新连接并刷新当前页面数据。
  - 切换失败时保持在原连接上，且不会被强制登出或跳回登录页。
  - （增强项）可选启用 `Ctrl+1~9` 快捷键切换，并提供关闭开关。

## 风险与待确认项

- 风险:
  - 管理密钥属于高敏数据：即使使用 `secureStorage` 加密，也仍落在浏览器端存储；需要明确“只适用于本机/本浏览器”的安全边界。
  - 如果实现登录后快速切换，可能涉及清理多个 store 缓存与路由状态，容易出现“切换后数据仍来自旧环境”的残留风险。
  - i18n 修复若只补 key，不做实际 UI 复现验证，可能漏掉其它菜单/按钮的缺失。

- 待确认:
