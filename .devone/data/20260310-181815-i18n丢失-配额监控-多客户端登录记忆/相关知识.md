## 术语与定义

- 客户端（Client Config）:
  - 一组用于连接某个 CLI Proxy API 实例的配置，至少包含：`name`、`apiBase`、`managementKey`。

- 活动客户端（Active Client）:
  - 当前被选中的客户端配置（`useClientCacheStore.activeClientId`）。
  - 设计上用于“默认填充登录信息 / 快速切换”。

- 记忆 / 持久化:
  - 客户端列表与活动客户端 ID 存在 `localStorage(client-cache)`。
  - 管理密钥单独存为 `secureStorage` 的加密项（避免明文写入 persist JSON）。

## 调研结论

- 结论 1：菜单多语言缺失的直接原因（已确认）
  - `MainLayout.tsx` 使用 `t('nav.monitor')`，但三种语言的 locale 文件 `nav` 均缺少 `monitor` key。

- 结论 2：多客户端“丢失”的直接原因（已确认）
  - 多客户端 store / UI / 快捷键 Hook 都已经存在，但当前没有任何页面引用它们；同时对应的 i18n key（`client_selector.*` / `client_management.*`）也未在 locale 文件中定义。

- 结论 3：配额菜单多语言
  - 当前 locale 的 `nav.quota_management` 在 `zh-CN/en/ru` 均存在；需要进一步复现你所说的“配额菜单丢失”的具体表现，判断是否为其它位置（例如页面标题/快捷入口）缺 key。
