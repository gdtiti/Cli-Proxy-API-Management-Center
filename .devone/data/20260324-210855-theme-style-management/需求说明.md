## 问题定义

- 当前问题:
  - 当前仓库只有一套官方原版视觉语言，主题切换仅覆盖 `light / white / dark / auto` 明暗语义，不能在保留原版风格的同时再接入一套新的界面风格。
  - 用户希望借鉴 `https://github.com/dear7575/Cli-Proxy-API-Management-Center.git` 的蓝灰、玻璃感、强化侧栏高亮的界面风格，但不希望覆盖掉现有官方原版样式。
- 影响对象:
  - `useThemeStore` 的持久化主题状态
  - `MainLayout` 顶栏主题入口
  - `themes.scss`、`layout.scss`、`components.scss` 的共享样式层
  - 依赖 `resolvedTheme` 的图标、图表和局部暗色适配逻辑
- 触发背景:
  - 需要做“主题风格管理”，而不是一次性的换肤覆盖。
  - 外部参考仓库已经提供了一套可复用的视觉 token 和布局皮肤，适合作为第二套风格族接入。
- 为什么现在处理:
  - 当前主题模型还是单轴结构，再继续叠加风格会导致状态含义混乱。
  - 现在先把“风格族”和“明暗模式”拆开，后续继续扩展第三套风格时不需要重做架构。

## 目标与成功标准

- 目标 1:
  - 保留现有官方原版风格和现有 `light / white / dark / auto` 使用体验，不让已有用户升级后被强制切换到新风格。
- 目标 2:
  - 新增一组参考仓库风格，能够通过管理界面切换，并在共享布局和基础组件上呈现出与参考仓库接近的视觉语言。
- 目标 3:
  - 主题状态应拆成“风格族 + 明暗模式”，并保持本地持久化与旧数据兼容。

## 范围边界

### 本次必须完成

- 为主题系统增加“风格族”状态，至少支持 `official` 与参考仓库风格两类。
- 保持现有官方主题可用，并确保旧持久化主题数据不报错。
- 在顶栏主题入口中提供风格族与明暗模式切换能力。
- 在共享样式层接入参考仓库风格的核心视觉差异：
  - 色板 token
  - 应用壳背景
  - 顶栏
  - 侧栏与导航激活态
  - 基础按钮、输入框、卡片、通知、模态框
- 补齐必要的国际化文案。
- 运行至少一组静态/构建验证，证明没有破坏现有前端构建。

### 本次明确不做

- 不逐页复刻参考仓库的全部业务样式差异。
- 不替换现有 logo、品牌文案、路由结构和功能布局。
- 不新增第三套以上风格族。
- 不为参考仓库风格单独做页面级定制资产或独立配置文件。

## 验收真相陈述

- [REQ-001] 现有官方原版风格仍可切换并保持原有 `light / white / dark / auto` 行为。
- [REQ-002] 新增的参考仓库风格可以在界面中被选择，并且用户刷新后仍能保留选择结果。
- [REQ-003] 参考仓库风格至少覆盖共享布局与基础组件层，不是只改主题菜单预览卡片。
- [REQ-004] 所有依赖 `resolvedTheme` 的现有暗色判断逻辑继续工作，不因风格族扩展而失效。

## 约束与依赖

- 技术约束:
  - 当前应用使用 React 19 + Zustand + SCSS，主题核心入口是 `src/stores/useThemeStore.ts`，主题 CSS 通过 `document.documentElement` 上的属性驱动。
  - 多处页面只关心 `resolvedTheme: 'light' | 'dark'`，因此扩展风格族时不能破坏这一派生值。
  - 当前 `Theme` 类型是 `'light' | 'white' | 'dark' | 'auto'`，需要在尽量小的改动范围内扩展状态模型。
- 业务约束:
  - 用户明确要求保留官方原版风格，不允许把现有视觉语言直接替换成参考仓库风格。
  - 这是管理后台，不允许为追求风格而牺牲信息可读性和交互可用性。
- 外部依赖:
  - 参考风格来源于 `dear7575/Cli-Proxy-API-Management-Center` 主分支 HEAD `facd59377df05d58f3003f2e389f5d08208c8d68`。

## 风险与待确认项

- 风险:
  - 参考仓库的视觉风格不仅在 `themes.scss`，还分布在 `layout.scss` 和 `components.scss`，如果接入范围过小，会变成“只有配色不像原版，整体风格又不完整”。
  - 官方风格支持 `white`，参考仓库风格没有对应纯白方案，需要定义清晰的兼容策略。
  - 状态模型变化涉及持久化数据，必须避免旧用户本地存储造成初始化异常。
- 待确认:
  - 本轮按默认假设推进：使用“风格族 + 明暗模式”模型；官方风格保留 `white`，参考仓库风格不单独提供 `white`，如当前模式为 `white` 且切到参考仓库风格，则显式规范化到 `light`。
