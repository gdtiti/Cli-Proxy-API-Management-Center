## 单元测试设计

### UT-001（构建/类型检查作为最小门禁）

- 目标: 保证 TS 编译与 bundle 构建通过
- 覆盖函数/模块: 全量（`tsc` + `vite build`）
- 输入: 当前工作区代码
- 预期输出: 命令返回码为 0 且生成 `dist/`
- 失败表现: `tsc` / `vite` 输出 error 或退出码非 0

## 属性测试设计

### PT-001（配置/JSON 的基本不变量）

- 性质: 本次修改涉及到的配置/JSON 必须保持“可解析 + 关键字段类型不变”
- 输入空间: 变更前后 JSON 文本
- 不变量:
  - `JSON.parse` 不抛错
  - key 为非空字符串
  - value 为字符串时保持字符串（例如路径映射）
- 预期失败模式: JSON 语法错误、意外引入环境耦合路径导致团队不可复现
