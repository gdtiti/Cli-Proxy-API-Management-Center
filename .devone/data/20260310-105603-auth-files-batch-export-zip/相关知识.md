## 术语与定义

- 认证文件（Auth File）: 管理端用于配置各类 AI Provider 凭证的 JSON 文件，文件名作为唯一标识。
- 标签（本次待确认）:
  - 可能指页面顶部的“类型筛选标签”（由 `file.type` 生成，例如 qwen/kimi/gemini/...）。
  - 也可能指“自定义 tags”（需要后端支持持久化与查询）。
- 筛选（Filter）: 当前页面对认证文件列表的过滤组合（类型/type + 搜索 + 问题过滤）。
- runtimeOnly: 运行态临时文件标记，不应参与批量操作（下载/删除/选中）。

## 调研结论

- 结论 1: “批量下载并打包 ZIP”的主能力在当前仓库已存在，若用户的“标签”=类型筛选标签，则需求大概率是“产品表达与入口优化”，而非从零开发。
- 结论 2: 若需要“自定义 tags 导出”，必须先确认后端是否已有 tags 字段与 API；仅前端无法可靠实现“按自定义 tags 导出”的数据来源与一致性。

## 风险假设

- 假设:
  - 用户所说“标签”可能=类型筛选标签（单选），因为当前 UI 已以标签形式呈现类型。
  - 认证文件数量可能达到几十/上百，当前“全量并发拉取”可能产生性能/稳定性风险。
- 验证方式:
  - 产品确认：明确“标签”的定义与期望交互（单标签、还是多标签、还是自定义 tags）。
  - 技术验证：在本地准备 50+ 文件进行批量导出，观察请求并发、浏览器内存与耗时；必要时加入并发上限。
