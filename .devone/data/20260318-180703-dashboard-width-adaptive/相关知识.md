# 相关知识

## 已知事实
- 主壳层 `MainLayout` 使用弹性布局，不存在统一的内容区 `max-width` 限制。
- 通用 `.container` 类虽有 `max-width: 1280px`，但当前 Dashboard 未直接使用该类。
- `DashboardPage` 的根容器是当前最直接的固定宽度来源。

## 调研结论
- 本次问题属于页面级样式回归，不是布局壳层回归。
- 修复只需落在 `DashboardPage.module.scss`。

## 风险假设
- 如果用户所说的“管理界面”还包含其他编辑页，则后续可能还会反馈其它页面的窄布局。
- 当前证据更集中在 Dashboard 首页，因此先按最小修复处理。
