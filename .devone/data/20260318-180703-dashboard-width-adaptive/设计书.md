# 设计书

## 方案摘要
- 采用最小修复。
- 保持 `MainLayout` 不变，只调整 `DashboardPage.module.scss` 的根容器样式。

## 设计依据
- `src/styles/layout.scss` 中：
  - `.content` 为 `flex: 1`
  - `.main-content` 为纵向弹性布局
- `src/pages/DashboardPage.module.scss` 中：
  - `.dashboard` 存在 `max-width: 1000px` 与 `margin: 0 auto`
  - 这是页面被锁定为窄栏的直接来源

## 锁定方案
- 删除 `.dashboard` 的固定宽度上限和居中收窄行为。
- 改为：
  - `width: 100%`
  - `min-width: 0`

## 备选方案对比
- 方案 A: 修改全局 `.main-content`
  - 风险: 会波及所有页面，回归面过大。
- 方案 B: 仅修 `DashboardPage`
  - 优点: 命中问题源头，变更最小，回归面可控。

## 设计结论
- 采用方案 B。
